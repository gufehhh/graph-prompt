### 🧠 Domain Knowledge (Crucial for Evaluation)

When evaluating the LLM's errors in the logs, enforce these strict topological definitions based on the solution space exhaustion:

1. **[Single Graph]**: A simple linked list connecting a starting point to an end point. Represents straightforward, single-logic problems (e.g., 1-to-n sequential steps solved by one continuous chain).
2. **[Concurrent Graphs]**: Multiple linked lists with entirely different starting and ending points. Represents concurrent logic problems (e.g., a prompt asking 2 to $n$ disconnected questions, where each answer forms a separate, non-intersecting linked list).
3. **[Divergent Graphs]**: Multiple linked lists that share the exact same starting point(s) and ending point(s), but utilize different intermediate paths. Represents a multi-perspective problem (e.g., one question with multiple valid methods/paths to reach the identical resolution).
4. **[Composite Graphs]**: A hybrid mixture of the above structures (e.g., combinations of concurrent and divergent linked lists within the same query).
5. **[Unclassifiable]**: Queries that are too ambiguous, broken, or lack the logical structure to be defined by the above topological graphs.
