## TOPOLOGY DEFINITIONS

| Category | Key Pattern | Decision Heuristic | Clear Examples | Common Mistakes |
|----------|-------------|-------------------|----------------|-----------------|
| Single Graph | Sequential chain | All reasoning steps connect in ONE path. Multi-hop is still Single. One question, one answer path. | LeetCode problems, math derivations, game simulations, round-based procedures, iterative algorithms, "Find X then use X to find Y" | ❌ Don't confuse internal algorithm conditions with Divergent |
| Concurrent Graphs | Parallel independent | Multiple questions with ZERO dependency. Each stands alone. Remove one, others still make sense. | "What is X? Also what is Y?" (no connection between X and Y), unrelated fact questions | ❌ Don't classify dependent multi-step as Concurrent |
| Divergent Graphs | Conditional branches | Query asks for DIFFERENT outcomes based on conditions. "What if A? What if B?" where A and B are separate scenarios. | "If price > $100 buy, else wait", "What happens if it rains vs if it's sunny?" (asks for both scenario outcomes) | ❌ Internal if-statements in algorithms ≠ Divergent |
| Composite Graphs | Mixed patterns | Has BOTH independent sub-queries AND sequential/conditional logic within those components. | "For each city, find X then Y. Also separately find Z." | ❌ Don't confuse complex Single with Composite |
| Unclassifiable | No clear structure | Too vague or ambiguous to map. Only for truly incoherent queries. | "tell me stuff", "what about that thing" | ❌ Complex but well-defined ≠ Unclassifiable |

## CRITICAL DISTINCTIONS (READ CAREFULLY)

### Single vs Concurrent
| Pattern | Classification | Why |
|---------|---------------|-----|
| "Find X, then use X to find Y" | Single Graph | Y depends on X - connected chain |
| "Find X. Also find Y." | Concurrent Graphs | X and Y independent - no connection |
| Question with context + single question | Single Graph | Context supports the question, not separate |
| Two unrelated questions | Concurrent Graphs | Can answer each without the other |

### Single vs Divergent (MOST IMPORTANT - READ TWICE)
| Pattern | Classification | Why |
|---------|---------------|-----|
| "Simulate a game where if player A wins, they get points" | Single Graph | One simulation, one outcome, internal rules |
| "Find the maximum element where x > 0" | Single Graph | Algorithm with constraint, one answer |
| "Round-based procedure with win conditions" | Single Graph | Single game simulation |
| "What if player A wins? What if player B wins?" | Divergent Graphs | Asks for TWO different scenario outcomes |
| "If price > $100, what should I do? If price ≤ $100, what then?" | Divergent Graphs | Explicitly asks for both branch outcomes |
| Math proof with cases leading to same conclusion | Single Graph | One conclusion, cases are internal steps |

**KEY RULE:** A single algorithm using if-statements internally = Single Graph (NOT Divergent). Divergent = query explicitly asks "what happens in scenario A vs scenario B?" expecting different answers for each.

### Concurrent vs Composite
| Pattern | Classification | Why |
|---------|---------------|-----|
| "What is X? What is Y? What is Z?" (all independent) | Concurrent Graphs | ALL sub-queries independent |
| "For each of X, Y, Z, find A then calculate B" | Single Graph | One complex process over multiple items |
| "Find X. Also, for Y, find A then B." | Composite Graphs | One independent + one sequential sub-query |

## WHAT IS ALWAYS SINGLE GRAPH (Common Mistakes)

- ✅ Game simulations with rules → Single Graph
- ✅ Algorithm problems with constraints → Single Graph  
- ✅ "Find X where condition Y is met" → Single Graph
- ✅ Round-based procedures → Single Graph
- ✅ Iterative/loop-based problems → Single Graph
- ✅ Math derivation problems (including those with LaTeX/formulas) → Single Graph
- ✅ Multi-hop reasoning where each step depends on previous → Single Graph
- ✅ Questions with setup/context followed by one question → Single Graph
- ❌ "Do A if X, do B if not X" (as separate scenario requests) → Divergent Graphs

## QUICK CLASSIFICATION FLOW

1. **Count questions**: How many distinct questions?
   - 1 question → Go to step 2
   - 2+ questions → Are they independent? Yes = Concurrent, No = check further

2. **Check dependencies**: Does any part depend on another part's answer?
   - Yes → Single Graph
   - No → Could be Concurrent or Divergent

3. **Scan for conditional words** (if/else/depending/cases/what if):
   - Is it asking "what happens in scenario A vs scenario B?" → Divergent Graphs
   - Is it a single algorithm/game with internal rules? → Single Graph

4. **Check for mixture**: Independent sub-queries with internal complexity?
   - Yes → Composite Graphs

5. **Check coherence**: Truly unclear/vague/incomplete?
   - Yes → Unclassifiable
   - No → Should be one of the above
