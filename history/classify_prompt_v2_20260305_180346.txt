3. `<topology_reasoning>`:

* Analyze the super-graph structure from Step 2 to determine the query's topological category.
* **CRITICAL GUIDANCE**: 
  - [Single Graph] should be used SPARINGLY - only for trivially simple queries with genuinely one solving method.
  - [Composite Graphs] is the DEFAULT for most non-trivial queries because different methods usually share some logic.
  - Distinguish carefully between Divergent (same start/end, completely different middles) vs Composite (shared intermediates OR convergent endpoints).
* **Topology Definitions with Concrete Examples**:
* **[Single Graph]**: One linear chain from start to end. ONLY for simple factual queries or direct calculations with no alternative approaches. **Use RARELY (< 5% of non-trivial queries).**
  - TRUE Examples: "What is 2+2?", "Who wrote Hamlet?", "What is the capital of France?", "Convert 5 miles to kilometers"
  - NOT Single: Any problem requiring multi-step reasoning, derivation, or analysis (these have multiple approaches)
* **[Concurrent Graphs]**: Two or more completely independent chains with different starts AND different ends. Use for multi-question prompts where questions are unrelated. **(~10-15% of queries)**
  - TRUE Examples: "What is 2+2? Also, explain quantum entanglement." (Unrelated topics)
  - TRUE Examples: "List the steps to bake a cake. Separately, what year did WWII end?" (No logical connection)
  - NOT Concurrent: "Given X, calculate Y. What if we change Z?" (Z modification depends on X→Y logic)
  - NOT Concurrent: Code + "Why does this fail? How to fix?" (Same code context, related questions)
* **[Divergent Graphs]**: Multiple chains with identical start AND identical end, but ZERO shared intermediate nodes. Each path uses entirely different intermediate logic. **Use RARELY (< 5%).** 
  - TRUE Examples: "Prove theorem X using three completely independent methods: (a) geometric construction, (b) algebraic manipulation, (c) probabilistic argument" where each method has no overlapping steps.
  - NOT Divergent: Math word problem solvable by algebra or geometry (both need to first define variables, set up equations → shared intermediates → Composite)
* **[Composite Graphs]**: Interconnected graph with shared intermediate nodes OR convergent endpoints. Includes: (a) paths branching then rejoining, (b) paths sharing common sub-routines/constraints, (c) hybrid structures mixing concurrent and divergent elements. **This is the MOST COMMON category (70-90% of non-trivial queries).**
  - TRUE Examples: Math problems (algebraic/geometric/combinatorial methods share intermediate steps)
  - TRUE Examples: Coding problems (multiple algorithms share input parsing or output formatting)
  - TRUE Examples: Analysis questions (multiple frameworks share data collection or premise analysis)
* **[Unclassifiable]**: Graph construction failed due to ambiguity.
* **Decision Process**: 
  1. First check: Multiple independent questions? → Concurrent
  2. Second check: Same start/end, zero shared intermediates? → Divergent  
  3. Third check: Genuinely one solving method? → Single
  4. Otherwise → Composite (default for reasoning-heavy queries)
* Provide explicit reasoning: Explain how the presence or absence of shared nodes and convergent paths led to your classification.
`</topology_reasoning>`
