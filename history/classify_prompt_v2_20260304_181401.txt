## TOPOLOGY DEFINITIONS

| Category | Key Pattern | Decision Heuristic | Examples |
|----------|-------------|-------------------|----------|
| Single Graph | Sequential chain | All reasoning steps connect in ONE path. Multi-hop is still Single. | LeetCode problems, math derivations, game simulations, round-based procedures, iterative algorithms |
| Concurrent Graphs | Parallel independent | Multiple questions with ZERO dependency. Each stands alone. | "What is X? Also what is Y?" (no connection between X and Y) |
| Divergent Graphs | Conditional branches | Contains if/then/else, "depending on", case-based logic where path splits. | "If X happens, do A; otherwise do B" (asks for different futures) |
| Composite Graphs | Mixed patterns | Has BOTH independent sub-queries AND sequential/conditional logic. | "For each city, find X then Y. Also separately find Z." |
| Unclassifiable | No clear structure | Too vague or ambiguous to map. | Only for truly incoherent queries like "tell me stuff" |

## CRITICAL DISTINCTIONS

**Single vs Concurrent:**
- Single: "Find X, then use X to find Y" → Connected chain
- Concurrent: "Find X. Also find Y." → Independent, no connection

**Single vs Divergent (MOST IMPORTANT):**
- Single: One path through the problem, even with internal conditions
- Divergent: Path splits based on conditions (asks for different outcomes)
- KEY RULE: A single algorithm using if-statements internally = Single Graph (NOT Divergent)
- ✅ "Simulate a game where if player A wins, they get points" → Single Graph (simulation)
- ✅ "Find the maximum element where x > 0" → Single Graph (algorithm with constraint)
- ✅ "Round-based procedure with win conditions" → Single Graph
- ❌ "What if player A wins? What if player B wins?" → Divergent Graphs (two scenarios)

**Concurrent vs Composite:**
- Concurrent: ALL sub-queries are independent
- Composite: SOME sub-queries are independent, but others have internal sequential/conditional logic

## WHAT IS ALWAYS SINGLE GRAPH (Common Mistakes)

- ✅ Game simulations with rules → Single Graph
- ✅ Algorithm problems with constraints → Single Graph  
- ✅ "Find X where condition Y is met" → Single Graph
- ✅ Round-based procedures → Single Graph
- ✅ Iterative/loop-based problems → Single Graph
- ✅ Math derivation problems → Single Graph
- ❌ "Do A if X, do B if not X" (as separate requests) → Divergent Graphs
