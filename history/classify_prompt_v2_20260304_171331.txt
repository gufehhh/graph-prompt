## TOPOLOGY DEFINITIONS

| Category | Key Pattern | Decision Heuristic |
|----------|-------------|-------------------|
| Single Graph | Sequential chain | All reasoning steps connect in ONE path. Multi-hop is still Single. |
| Concurrent Graphs | Parallel independent | Multiple questions with ZERO dependency. Each stands alone. |
| Divergent Graphs | Conditional branches | Contains if/then/else, "depending on", case-based logic. |
| Composite Graphs | Mixed patterns | Has BOTH independent sub-queries AND sequential/conditional logic. |
| Unclassifiable | No clear structure | Too vague or ambiguous to map. |

## CRITICAL DISTINCTIONS

**Single vs Concurrent:**
- Single: "Find X, then use X to find Y" → Connected chain
- Concurrent: "Find X. Also find Y." → Independent, no connection

**Single vs Divergent:**
- Single: One path through the problem
- Divergent: Path splits based on conditions (look for: if, else, when, depending)

**Concurrent vs Composite:**
- Concurrent: ALL sub-queries are independent
- Composite: SOME sub-queries are independent, but others have internal sequential/conditional logic
