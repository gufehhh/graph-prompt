3. `<topology_reasoning>`:

* Analyze the super-graph structure from Step 2 to determine the query's topological category.
* **CRITICAL GUIDANCE**: 
  - Avoid classification bias. Do NOT default to any single category.
  - Follow the decision tree STRICTLY in order. Stop at the first TRUE match.
  - Each category has specific structural requirements - verify ALL are met.
* **TOPOLOGY DEFINITIONS WITH STRUCTURAL CRITERIA AND EXAMPLES**:

* **[Concurrent Graphs]** - MULTIPLE INDEPENDENT QUESTIONS (~10-15% of queries):
  - **STRUCTURAL CRITERIA**: 
    - 2+ disconnected sub-graphs with NO shared nodes
    - Different starting points AND different ending points
    - Zero logical connection between sub-graphs
  - **TRUE Examples**: 
    - "What is 2+2? Also, explain quantum entanglement." (Unrelated topics, separate papers)
    - "List the steps to bake a cake. Separately, what year did WWII end?" (No logical connection)
    - "What's the weather in Tokyo? Also, who wrote Pride and Prejudice?" (Completely independent)
  - **FALSE Examples (NOT Concurrent)**:
    - "Given X, calculate Y. What if we change Z?" (Z modification depends on X→Y logic, shares context)
    - Code + "Why does this fail? How to fix?" (Same code context, related questions)
    - "Calculate the area. Then calculate the volume using the area." (Sequential dependency)
    - Multi-part math problem where part (b) builds on part (a)
  - **DECISION TEST**: "Can I answer each question completely independently without referencing the other?" If NO → NOT Concurrent.

* **[Divergent Graphs]** - SAME START/END, COMPLETELY DIFFERENT MIDDLES (<5% of queries):
  - **STRUCTURAL CRITERIA**:
    - Exactly ONE starting node
    - Exactly ONE ending node
    - ZERO shared intermediate nodes across paths (each path uses 100% unique intermediates)
  - **TRUE Examples**:
    - "Prove theorem X using three completely independent methods: (a) geometric construction with no algebra, (b) pure algebraic manipulation with no geometry, (c) combinatorial argument with no geometry or algebra" where each method has ZERO overlapping intermediate steps.
    - "What are 3 completely unrelated routes from city A to city B: (a) flight (uses airports, airlines), (b) train (uses rail infrastructure), (c) ship (uses ports, waterways)" - no shared infrastructure.
  - **FALSE Examples (NOT Divergent)**:
    - Math word problem solvable by algebra or geometry (both need to first define variables, set up equations → shared "define variables" intermediate → Composite)
    - Coding problem with multiple algorithms (all share input parsing, variable initialization, or output formatting → Composite)
    - Any problem where different methods share preprocessing, setup, or common sub-routines
  - **CRITICAL DECISION TEST**: "Do at least 2 paths share ANY intermediate step, constraint, variable definition, or sub-routine?" If YES → NOT Divergent, it's Composite.

* **[Single Graph]** - ONE LINEAR CHAIN (<5% of queries, use SPARINGLY):
  - **STRUCTURAL CRITERIA**:
    - Exactly ONE starting node
    - Exactly ONE ending node
    - ONE linear sequence of intermediates with no branching
    - Only ONE viable solving method exists
  - **TRUE Examples** (Simple factual recall or direct calculation ONLY):
    - "What is 2+2?", "Who wrote Hamlet?", "What is the capital of France?"
    - "What year did WWII end?", "What is the atomic number of carbon?"
    - "Convert 5 miles to kilometers", "What is 100 divided by 4?"
  - **FALSE Examples (NOT Single)**:
    - ANY problem requiring multi-step derivation, reasoning, or analysis
    - Math word problems (have multiple solution approaches)
    - Coding problems (have multiple debugging/implementation strategies)
    - Analysis questions (have multiple analytical frameworks)
    - "Explain X" where X is a complex concept (has multiple explanatory approaches)
  - **CRITICAL DECISION TEST**: "Could a reasonable person solve this using a different valid approach?" If YES → NOT Single.

* **[Composite Graphs]** - INTERCONNECTED/HYBRID STRUCTURES (70-90% of non-trivial queries):
  - **STRUCTURAL CRITERIA** (meet ANY of these):
    - Multiple paths share at least ONE intermediate node
    - Multiple paths converge on the same ending point
    - Paths branch from common start and rejoin before the end
    - Hybrid mixture of parallel and sequential elements
    - Different methods share common sub-routines, constraints, or preprocessing
  - **TRUE Examples** (Most non-trivial queries):
    - Math problems (algebraic/geometric/combinatorial methods share intermediate steps like "define variables" or "set up equations")
    - Coding problems (multiple algorithms share input parsing, variable initialization, or output formatting)
    - Analysis questions (multiple frameworks share data collection, premise analysis, or conclusion synthesis)
    - Debugging problems (multiple diagnostic approaches share code inspection or log analysis)
  - **DECISION RULE**: If you have multiple solving paths AND they share ANY intermediate logic, constraint, or sub-routine → Composite.

* **[Unclassifiable]** - Graph construction failed due to extreme ambiguity.

* **MANDATORY DECISION PROCESS** (Follow in exact order):
  1. **CONCURRENT CHECK**: Multiple disconnected sub-graphs with different starts AND different ends? → [Concurrent Graphs]
  2. **DIVERGENT CHECK**: Same start, same end, ZERO shared intermediates? → [Divergent Graphs]
  3. **SINGLE CHECK**: Genuinely one solving method only (simple factual recall)? → [Single Graph]
  4. **DEFAULT**: None of the above → [Composite Graphs]
